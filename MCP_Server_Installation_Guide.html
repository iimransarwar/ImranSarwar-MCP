This guide will walk you through installing and configuring the MCP (Model Context Protocol) Server for Salesforce integration with Claude Desktop.

üìã Prerequisites

Before beginning, ensure you have:

  • Windows operating system (for this guide)
  • Administrator privileges on your computer
  • Salesforce credentials (username, password, and security token)
  • Claude Desktop application installed

Step 1: Install Node.js

Node.js is required to run the MCP server. We'll install the LTS (Long Term Support) version for stability.

1Download Node.js LTS

  • Navigate to: https://nodejs.org/dist/v22.18.0/node-v22.18.0-x64.msi
  • Download the installer file

2Run the installer

  • Double-click the downloaded .msi file
  • Follow the installation wizard prompts
  • Accept the default installation options
  • Click "Install" and wait for completion

3Verify Node.js installation

Open Command Prompt (Windows Key + R, type cmd, press Enter) and run:

node --version npm --version

You should see version numbers for both Node.js and npm

Step 2: Install Salesforce CLI

The Salesforce CLI provides essential tools for interacting with Salesforce orgs.

1Install Salesforce CLI globally

npm install -g @salesforce/cli

2Verify Salesforce CLI installation

sf --version

This should display the Salesforce CLI version

Step 3: Configure npm Authentication

1Login to npm

In your terminal/Command Prompt, type:

npm login
  • This will provide you with a login URL in the command line
  • Copy this URL and share it with your administrator (Imran) for authentication

2Confirm login

Once authenticated, you'll see a confirmation message in your terminal indicating successful login

Step 4: Install MCP Server Package

1Install the MCP Server package

npm i @iimransarwar/mcp-server-salesforce

Wait for the installation to complete

2Run npm fund (optional)

npm fund

This displays funding information for installed packages

Step 5: Configure Claude Desktop

1Access Claude Desktop Settings

  • Open Claude Desktop application
  • Navigate to Settings
  • Look for "MCP Servers" section
  • Click the "Edit Config" button

2Locate configuration file

  • The Edit Config button will open the Claude configuration folder
  • Find the file named claude_desktop_config.json

3Edit configuration file

Open claude_desktop_config.json in a text editor and replace the entire contents with:

{ "mcpServers": { "salesforce": { "command": "npx", "args": ["-y", "@iimransarwar/mcp-server-salesforce"], "env": { "SALESFORCE_USERNAME": "your.username@example.com", "SALESFORCE_PASSWORD": "YourPassword", "SALESFORCE_TOKEN": "YourSecurityToken", "SALESFORCE_INSTANCE_URL": "https://login.salesforce.com" } } } }

4Update credentials

Replace the placeholder values with your actual Salesforce credentials:

  • SALESFORCE_USERNAME: Your Salesforce username
  • SALESFORCE_PASSWORD: Your Salesforce password
  • SALESFORCE_TOKEN: Your Salesforce security token
  • SALESFORCE_INSTANCE_URL: Your Salesforce org URL (use https://test.salesforce.com for sandbox orgs)

5Save the configuration

  • Save the claude_desktop_config.json file
  • Close the text editor

Step 6: Restart Claude Desktop

1Close Claude Desktop completely

  • Ensure Claude is fully closed (check system tray)

2Reopen Claude Desktop

  • Launch Claude Desktop again
  • The MCP Server should now be active and connected to your Salesforce org

‚úÖ Verification

To verify the installation:

  • In Claude, you should now have access to Salesforce-related capabilities
  • The MCP server icon should appear in Claude's interface
  • Try a simple Salesforce query to confirm the connection is working

Switching Between Salesforce Orgs

To connect to a different Salesforce org in the future:

  1. Navigate back to the claude_desktop_config.json file
  2. Update the credentials in the environment variables section
  3. Save the file
  4. Restart Claude Desktop for changes to take effect

üîß Troubleshooting Tips

  • If Node.js commands aren't recognized: Ensure Node.js is properly added to your system PATH
  • If npm login fails: Check your internet connection and proxy settings
  • If Claude doesn't recognize the MCP server: Ensure the JSON configuration is properly formatted (no missing commas or brackets)
  • For Salesforce connection issues: Verify your security token is correct and appended to your password if required

üîí Security Notes

  • Keep your claude_desktop_config.json file secure as it contains sensitive credentials
  • Consider using environment variables for production deployments
  • Regularly rotate your Salesforce security tokens
  • Never commit the configuration file to version control systems

Your MCP Server for Salesforce is now successfully installed and configured! You can begin using Claude with enhanced Salesforce capabilities.
